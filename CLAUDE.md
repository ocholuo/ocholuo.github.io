## Repository Overview

This is a Jekyll-based personal blog (https://ocholuo.github.io) using the [Chirpy theme](https://github.com/cotes2333/jekyll-theme-chirpy). It's a cybersecurity-focused blog with 23+ post categories covering security, cloud, networking, AI/ML, and lab writeups.

## Development Commands

### Setup

```bash
# Python virtual environment (for pre-commit hooks)
pyenv install 3.11.4
pyenv virtualenv 3.11.4 ocho
pyenv activate ocho
brew install pre-commit
pre-commit install -t pre-push
```

### Local Development

```bash
brew install rbenv ruby-build
echo 'eval "$(rbenv init -)"' >> ~/.zshrc
source ~/.zshrc

brew install openssl readline libyaml gmp
RUBY_CONFIGURE_OPTS="--with-openssl-dir=$(brew --prefix openssl@3)" \
rbenv install 3.2.2

rbenv init
rbenv local 3.2.2
rbenv rehash
which ruby
# ~/.rbenv/shims/ruby   ✅
# /usr/bin/ruby         ❌
ruby -v
# ruby 3.2.2

# Ruby/Jekyll dependencies
gem install bundler
bundle install

# Node.js dependencies (for JS/CSS asset pipeline)
npm install
```

```bash
# Build production assets first (required before jekyll serve on fresh clone)
npm run build

# Watch JS/CSS assets during development
npm run watch

# Serve site locally with live reload
bundle exec jekyll serve
```

### Testing & Linting

```bash

```

## Post Structure and Conventions

### File Naming

Posts live in `_posts/` and follow this pattern:
```
_posts/<category>/YYYY-MM-DD-title-with-dashes.md
```

### Front Matter

```yaml
---
title: "Post Title"
date: 2024-01-01 10:00:00
categories: [TopLevel, SubLevel]
tags: [tag1, tag2]
toc: true          # Table of contents
pin: false         # Pin to top
math: false        # Enable math rendering
mermaid: false     # Enable Mermaid diagrams
---
```

### Category Directories

Posts are organized in numbered directories for sorting:
- `00Basic/`, `00CodeNote/` — Fundamentals and code notes
- `01Cloud/` — Cloud (AWS, Azure, GCP)
- `02Security/`, `10CyberAttack/`, `10SecConcepts/` — Security
- `13Crypt/`, `14AppSec/`, `15NetworkSec/`, `16NetworkSecII/` — Security specialties
- `18DefendSec/` — Defensive security
- `30System/` — System administration
- `50BigData/`, `51AI/` — Data and AI/ML
- `99Lab/`, `Lab/` — Lab writeups (SEED Labs, HTB, OverTheWire, etc.)
- `SOC/`, `Web/`, `certification/` — SOC, web, certifications

## Architecture

### Theme and Build Pipeline

- **Jekyll theme**: `jekyll-theme-chirpy` — layouts, includes, and SCSS are inherited from the gem; local files in `_layouts/`, `_includes/`, `_sass/` override the theme
- **JS bundling**: Rollup (`_javascript/` → `assets/js/dist/`) with Babel for ES6 transpilation
- **CSS**: SCSS in `_sass/` compiled by Jekyll
- **Assets**: Compiled JS output goes to `assets/js/dist/` (not committed; generated by `npm run build`)

### Key Configuration

`_config.yml` controls site-wide settings. Notable fields:
- `baseurl` — empty (served at root)
- `comments.disqus.shortname` — Disqus integration
- `analytics.google.id` — Google Analytics
- `pwa.enabled: true` — Progressive Web App with offline support

### CI/CD (`.github/workflows/`)

- `ci.yml` — Runs `./tools/test` on PRs
- `pages-deploy.yml` — Deploys to GitHub Pages on push to `main`
- `codeql.yml` — Security scanning

## Code Quality

Pre-commit hooks run automatically on push:
- YAML/JSON validation
- Private key detection
- Trailing whitespace cleanup
- Python: autoflake, isort

Commit messages are validated by CommitLint (see `.commitlintrc.json`).

Markdown style is enforced by `.markdownlint.json`.
